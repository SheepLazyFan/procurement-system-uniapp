#### API 请求层设计要点

- **统一封装：** `request.js` 基于 `uni.request` 封装，自动注入 `Authorization: Bearer <token>`
- **白名单：** `/auth/login`、`/auth/sms/send`、`/buyer/` 开头的公开接口无需 Token
- **错误处理：** 业务码 `code !== 200` 弹 Toast，HTTP 401 自动跳转登录页
- **便捷方法：** 导出 `get`、`post`、`put`、`del`、`uploadFile` 五个函数
- **BASE_URL：** 开发环境 `http://localhost:8080/api`

#### 状态管理设计

| Store | 关键状态 | 核心功能 |
|---|---|---|
| `user` | token, userInfo, isLoggedIn | 登录/登出、角色判断(isSeller/isMember/isBuyer)、Token 持久化 |
| `enterprise` | info, loaded | 企业信息 CRUD、邀请码刷新 |
| `cart` | enterpriseId, items | 多企业隔离购物车、商品增删改、库存校验、localStorage 持久化 |
| `printer` | connectedDevice, discoveredDevices | 蓝牙设备管理、纸宽设置、连接状态恢复 |
