## 1. 项目概述

### 1.1 产品定位

面向中小型商家的 **B2B 采购管理微信小程序**。商家可在小程序内创建企业、管理商品库存，并通过微信社交链路将商品分享给客户，客户可直接在小程序内完成采购下单。商家端同时集成管理后台功能，实现"一个小程序，既是店铺也是管理平台"。

### 1.2 核心业务逻辑

```
商家注册 → 创建企业 → 添加商品分类/商品 → 分享库存给客户
                                                    ↓
客户点击分享链接 → 进入商家库存页 → 浏览商品 → 采购下单（此时授权微信）
                                                    ↓
商家收到订单 → 处理销售订单 → 库存自动更新 → 查看统计报表
```

### 1.3 目标用户

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| **商家（卖家）** | 中小型企业主/采购负责人 | 管理商品、处理订单、查看统计 |
| **团队成员** | 商家邀请的员工 | 协助管理库存和订单 |
| **买家（客户）** | 商家的下游客户 | 浏览商品、下单采购 |

## 2. 技术栈

### 2.1 前端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端框架** | Vue 3 + uni-app (Vite 构建) | 跨平台开发，优先适配微信小程序 |
| **UI 框架** | uni-ui + 自定义组件 | 官方组件库，兼容性好 |
| **状态管理** | Pinia | Vue 3 官方推荐状态管理 |
| **网络请求** | uni.request 封装 (拦截器 + Token) | 统一请求/响应处理，自动携带鉴权 Token |
| **Excel 解析** | xlsx (SheetJS) | 前端解析 Excel 文件（批量导入预览） |
| **蓝牙打印** | 微信蓝牙 API + ESC/POS 协议 | 连接热敏打印机，打印纸质订单小票 |

### 2.2 后端技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **后端框架** | Java 17 + Spring Boot 3.x | 主流企业级后端框架，长期支持版本 |
| **ORM 框架** | MyBatis-Plus | 简化 CRUD，内置分页/逻辑删除/自动填充 |
| **数据库** | MySQL 8.x | 关系型数据库，成熟稳定 |
| **数据库连接池** | HikariCP | Spring Boot 默认连接池，性能最优 |
| **认证鉴权** | Spring Security + JWT | 无状态 Token 认证，适配小程序场景 |
| **参数校验** | Jakarta Validation (Hibernate Validator) | 后端入参双重校验 |
| **接口文档** | Knife4j (Swagger 增强) | 在线 API 文档，方便前后端联调 |
| **文件存储** | 腾讯云 COS (对象存储) | 商品图片、Excel 模板、数据备份文件，与微信生态同网络链路 |
| **缓存** | Redis | 验证码缓存、Token 黑名单、热点数据缓存 |
| **消息推送** | 微信订阅消息 (服务端 API) | 通过后端调用微信接口推送订单通知 |
| **短信服务** | 腾讯云短信 | 商家手机号登录验证码 |
| **Excel 处理** | EasyExcel (阿里开源) | 后端解析/生成 Excel（批量导入写入、数据备份导出） |
| **日志框架** | SLF4J + Logback | Spring Boot 默认日志，结构化日志输出 |

### 2.3 基础设施

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **服务器** | 腾讯云轻量应用服务器 | 2核4G 起步，部署 Spring Boot + MySQL + Redis |
| **反向代理** | Nginx | HTTPS 证书、请求转发、静态资源代理 |
| **部署方式** | JAR 包直接部署 / Docker（可选） | 简单运维，后续可升级容器化 |
| **域名 & SSL** | 腾讯云域名 + 免费 DV SSL 证书 | 小程序要求后端必须 HTTPS |
| **微信对接** | 微信小程序服务端 SDK | 登录凭证校验、订阅消息、小程序码生成（同腾讯生态，网络链路更短） |

### 2.4 前后端交互方式

```
前端 (uni-app 小程序)
    │
    │  HTTPS 请求 (RESTful API)
    │  Header: Authorization: Bearer <JWT Token>
    ↓
Nginx (反向代理 + SSL 终止)
    │
    ↓
Spring Boot 后端 (端口 8080)
    ├── Spring Security 过滤链 → JWT 解析 → 用户身份识别
    ├── Controller → Service → Mapper (MyBatis-Plus)
    ├── MySQL 8.x (业务数据)
    ├── Redis (缓存 + 验证码 + Token)
    └── 腾讯云 COS (文件存储)
```

## 4. 用户故事与验收标准

### US-01：商家注册与登录

> **作为**商家用户，**我希望**通过手机号注册和登录小程序，**以便**管理我的企业和商品。

**验收标准 (AC)：**
- [ ] AC-01.1：用户输入手机号，点击获取验证码，60秒内收到短信验证码
- [ ] AC-01.2：输入正确验证码后成功登录，跳转至"我的"页面
- [ ] AC-01.3：首次登录的用户，"我的"页面显示"未创建企业"状态
- [ ] AC-01.4：已登录用户再次打开小程序自动保持登录状态
- [ ] AC-01.5：登录态过期后提示重新登录

### US-02：创建与管理企业

> **作为**商家用户，**我希望**在小程序内创建和编辑我的企业信息，**以便**展示给客户。

**验收标准 (AC)：**
- [ ] AC-02.1：未创建企业的用户可以填写企业名称、联系方式等信息创建企业
- [ ] AC-02.2：创建成功后，"我的"页面显示企业信息和管理入口
- [ ] AC-02.3：可随时编辑企业名称、地址、联系方式等基本信息
- [ ] AC-02.4：企业信息变更后，分享给客户的页面同步更新

### US-03：商品分类管理

> **作为**商家用户，**我希望**自定义商品分类，**以便**结构化地组织我的商品。

**验收标准 (AC)：**
- [ ] AC-03.1：可以创建、编辑、删除商品分类
- [ ] AC-03.2：分类支持自定义名称和排序
- [ ] AC-03.3：删除分类时，若该分类下有商品，需提示用户先移动或删除商品
- [ ] AC-03.4：分类列表按排序值展示

### US-04：商品管理

> **作为**商家用户，**我希望**添加和管理商品信息（名称、规格、价格、库存等），**以便**维护我的库存。

**验收标准 (AC)：**
- [ ] AC-04.1：可以添加商品，填写名称、分类、规格、单位、售价、成本价、库存数量、商品图片
- [ ] AC-04.2：商品列表支持按分类筛选
- [ ] AC-04.3：可以编辑和删除已有商品
- [ ] AC-04.4：库存数量支持手动调整（入库/出库）
- [ ] AC-04.5：商品图片支持上传至云存储，最多可上传多张
- [ ] AC-04.6：库存预警 — 商品可设置库存预警阈值，低于阈值时在库存首页醒目标识（如红色提示）

### US-05：库存分享

> **作为**商家用户，**我希望**将我的商品分类和库存信息通过微信分享给客户，**以便**客户直接浏览和采购。

**验收标准 (AC)：**
- [ ] AC-05.1：在库存页面可以选择"分享给客户"，生成小程序分享卡片
- [ ] AC-05.2：分享卡片包含商家名称、商品概览信息
- [ ] AC-05.3：客户点击分享卡片直接进入该商家的库存浏览页
- [ ] AC-05.4：可以分享整个库存或按分类分享特定商品

### US-06：买家浏览与采购

> **作为**买家，**我希望**通过商家分享的链接直接浏览商品并下单采购，**以便**快捷地完成进货。

**验收标准 (AC)：**
- [ ] AC-06.1：点击分享链接免登录即可浏览商家的商品列表和分类
- [ ] AC-06.2：可以查看商品详情（名称、规格、价格、库存状态）
- [ ] AC-06.3：可以将商品加入采购清单（购物车），自由调整数量
- [ ] AC-06.4：提交订单时需要微信授权（获取头像、昵称）
- [ ] AC-06.5：点击"支付"按钮后，直接跳转至支付成功页面（伪支付流程，不接入真实微信支付）
- [ ] AC-06.6：提交订单后，商家端实时收到新订单通知
- [ ] AC-06.7：买家可查看自己的历史采购订单和订单状态

### US-07：销售订单管理

> **作为**商家用户，**我希望**管理客户的采购订单（销售订单），**以便**跟踪订单状态和处理发货。

**验收标准 (AC)：**
- [ ] AC-07.1：销售订单列表展示所有客户的采购订单，支持按状态筛选（待确认/已确认/已发货/已完成）
- [ ] AC-07.2：可以查看订单详情：客户信息、商品明细、总金额
- [ ] AC-07.3：可以确认订单、标记发货、完成订单
- [ ] AC-07.4：支持"开单"功能 — 商家主动为客户创建销售订单（线下场景）
- [ ] AC-07.5：订单状态变更后，库存数量自动扣减/恢复
- [ ] AC-07.6：订单支付状态显示（伪支付：已支付/未支付，第一期点击支付即标记为已支付）
- [ ] AC-07.5：订单状态变更后，库存数量自动扣减/恢复

### US-08：客户管理

> **作为**商家用户，**我希望**管理我的客户信息，**以便**维护客户关系和查看交易记录。

**验收标准 (AC)：**
- [ ] AC-08.1：自动记录通过小程序下单的客户信息
- [ ] AC-08.2：支持手动添加新客户（名称、联系方式、地址）
- [ ] AC-08.3：客户详情页展示该客户的历史订单记录
- [ ] AC-08.4：可以编辑和删除客户信息

### US-09：采购订单管理（商家向上游进货）

> **作为**商家用户，**我希望**记录和管理我向供应商的采购订单，**以便**追踪进货情况。

**验收标准 (AC)：**
- [ ] AC-09.1：支持"快速采购"功能 — 从库存中选择商品，快速生成采购订单
- [ ] AC-09.2：采购订单列表支持按状态筛选（待采购/采购中/已到货/已完成）
- [ ] AC-09.3：采购到货后，库存自动增加
- [ ] AC-09.4：可以查看采购订单详情和历史记录

### US-10：供应商管理

> **作为**商家用户，**我希望**管理我的供应商信息，**以便**快速采购和追踪进货来源。

**验收标准 (AC)：**
- [ ] AC-10.1：支持添加供应商（名称、联系方式、地址、主营品类）
- [ ] AC-10.2：供应商详情页展示关联的采购记录
- [ ] AC-10.3：可以编辑和删除供应商信息
- [ ] AC-10.4：快速采购时可以选择关联供应商

### US-11：统计报表

> **作为**商家用户，**我希望**查看经营数据统计（销售额、采购额、库存概况），**以便**做出经营决策。

**验收标准 (AC)：**
- [ ] AC-11.1：统计首页展示关键指标：今日销售额、本月销售额、库存总值、待处理订单数
- [ ] AC-11.2：支持按时间范围（日/周/月）查看销售趋势图表
- [ ] AC-11.3：库存统计展示各分类商品数量和库存预警（低于阈值商品数）
- [ ] AC-11.4：支持销售排行榜（按商品/按客户）
- [ ] AC-11.5：利润统计 — 展示毛利润（销售额 - 成本），支持按日/周/月查看利润趋势
- [ ] AC-11.6：利润明细 — 可查看每笔订单的利润（售价 × 数量 - 成本价 × 数量）

### US-12：团队管理

> **作为**企业主，**我希望**邀请团队成员共同管理企业，**以便**分担日常运营工作。

**验收标准 (AC)：**
- [ ] AC-12.1：企业主可以生成邀请码或邀请链接
- [ ] AC-12.2：被邀请人通过手机号登录后，输入邀请码加入企业
- [ ] AC-12.3：企业主可以设置团队成员权限（如仅允许管理库存）
- [ ] AC-12.4：企业主可以移除团队成员

### US-13：数据备份

> **作为**企业主，**我希望**备份和导出企业数据，**以便**防止数据丢失。

**验收标准 (AC)：**
- [ ] AC-13.1：支持一键备份全部企业数据（商品、订单、客户等）
- [ ] AC-13.2：展示备份历史记录和时间
- [ ] AC-13.3：支持从备份恢复数据

### US-14：蓝牙打印订单

> **作为**商家用户，**我希望**通过蓝牙连接热敏打印机，一键打印纸质订单小票，**以便**快速出具纸质凭证给客户或留档。

**验收标准 (AC)：**
- [ ] AC-14.1：在"我的"页面提供"蓝牙打印"入口，进入蓝牙打印机管理页
- [ ] AC-14.2：支持搜索附近蓝牙打印设备，展示设备列表（名称、信号强度、连接状态）
- [ ] AC-14.3：支持连接/断开蓝牙打印机，连接成功后显示已连接状态
- [ ] AC-14.4：已连接打印机时，在销售订单详情页和采购订单详情页显示"打印订单"按钮
- [ ] AC-14.5：打印的纸质小票包含以下关键信息：企业名称、订单号、订单创建时间、商品明细（品名、单位、数量、单价、金额）、总金额、客户/供应商名称
- [ ] AC-14.6：打印前可预览小票内容
- [ ] AC-14.7：打印失败时给出明确的错误提示（蓝牙未开启、打印机断开、缺纸等）
- [ ] AC-14.8：支持记住上次连接的打印机，下次自动尝试重连
- [ ] AC-14.9：默认使用 80mm 纸张宽度排版（表头列：品名、单位、数量、单价、金额）
- [ ] AC-14.10：支持自适应打印 — 自动检测/手动选择纸张宽度（58mm / 80mm），根据纸宽动态调整列宽和字号
- [ ] AC-14.11：58mm 模式下，对列进行智能缩排（如品名截断省略、合并单位列等），确保关键信息完整不溢出
- [ ] AC-14.12：打印设置页可配置纸张宽度偏好（58mm / 80mm），配置保存至打印机记录中

### US-15：批量导入商品

> **作为**商家用户，**我希望**通过上传 Excel 表格批量导入商品分类和商品信息，**以便**快速完成大量商品的上架。

**验收标准 (AC)：**
- [ ] AC-15.1：在库存首页提供"批量导入"按钮
- [ ] AC-15.2：提供 Excel 模板下载功能，模板包含字段说明（分类名称、商品名称、规格、单位、价格、库存数量）
- [ ] AC-15.3：支持从微信聊天记录或手机文件中选择 Excel 文件（.xlsx / .xls）

## 7. 核心业务流程

### 7.1 商家注册与建店流程

```
用户打开小程序
    ↓
显示登录页 → 输入手机号 → 获取验证码 → 验证成功
    ↓
进入"我的"页面（未创建企业状态）
    ↓
点击"创建企业" → 填写企业名称/地址/联系方式 → 提交
    ↓
创建成功 → 页面更新为"已创建企业"状态
    ↓
开始添加商品分类 → 添加商品 → 管理库存
```

### 7.2 库存分享与买家采购流程

```
商家在库存页面点击"分享"
    ↓
选择分享范围（全部/按分类）→ 生成小程序分享卡片
    ↓
通过微信聊天/群分享给客户
    ↓
客户点击卡片 → 进入该商家的库存浏览页（免登录）
    ↓
浏览商品 → 加入采购清单 → 调整数量
    ↓
点击"提交订单" → 微信授权登录 → 填写收货信息 → 确认提交
    ↓
点击"支付" → 直接跳转支付成功页（伪支付，第一期不接入真实支付）
    ↓
商家端收到新订单通知 → 销售订单列表出现新订单
```

### 7.3 销售订单处理流程

```
                     ┌─ 买家线上下单
新订单来源 ─────────┤
                     └─ 商家手动"开单"（线下场景）
    ↓
订单状态：待确认 → 商家确认订单
    ↓
订单状态：已确认 → 商家备货/发货 → 标记发货
    ↓
订单状态：已发货 → 客户确认收货
    ↓
订单状态：已完成 → 库存自动扣减 → 计入统计
                                    ↓（可选）
                          商家点击"打印订单" → 蓝牙发送至打印机 → 输出纸质小票
```

### 7.4 采购订单流程（商家向上游进货）

```
商家进入采购订单模块 → 点击"快速采购"
    ↓
从库存中选择需要补货的商品 → 设置采购数量
    ↓
选择/新建供应商 → 生成采购订单
    ↓
订单状态：待采购 → 采购中 → 已到货
    ↓
确认到货 → 库存自动增加 → 计入统计
```

### 7.5 蓝牙打印流程

```
商家进入"我的" → 点击"蓝牙打印"
    ↓
开启蓝牙适配器 → 搜索附近蓝牙打印设备
    ↓
设备列表展示（名称 + 信号强度）→ 选择目标打印机 → 连接
    ↓
连接成功 → 设置纸张宽度偏好（默认80mm / 可选58mm）→ 保存设备信息
    ↓
返回订单详情页 → 点击"打印订单"
    ↓
根据纸宽设置自适应排版（80mm 完整五列 / 58mm 智能缩排）
    ↓
生成 ESC/POS 打印指令（企业名称、订单号、时间、商品明细表：品名|单位|数量|单价|金额、总价）
    ↓
通过 BLE 发送数据 → 打印机输出纸质小票
```

### 7.6 批量导入商品流程

```
商家进入库存首页 → 点击"批量导入"
    ↓
选择"下载模板" → 获取标准 Excel 模板
    或
选择"上传文件" → 从微信聊天/手机文件选择 Excel
    ↓
前端解析 Excel 文件（xlsx 库）→ 展示预览页面
    ↓
数据校验：
  ✅ 合格行（绿色标记）
  ❌ 不合格行（红色标记 + 错误原因）
    ↓
同名商品处理策略选择：跳过 / 覆盖更新
    ↓
用户确认 → 调用后端接口批量写入
    ↓
自动创建不存在的分类 → 批量插入商品数据
    ↓
展示导入结果报告：成功 N 条、失败 M 条、新建分类 X 个
```

## 8. 非功能性需求

### 8.1 安全性

| 需求项 | 说明 |
|--------|------|
| **数据隔离** | 每个企业的数据严格隔离，所有查询自动拼接 `enterprise_id` 条件，商家只能访问自己企业的数据 |
| **接口鉴权** | 所有 API 通过 Spring Security + JWT 验证用户身份和权限，Token 有效期 7 天 |
| **敏感数据** | 手机号等敏感信息加密存储，密码使用 BCrypt 哈希，不在前端明文展示完整手机号 |
| **输入校验** | 所有用户输入在前端和后端双重校验（Jakarta Validation），防止 SQL 注入和 XSS 攻击 |
| **分享安全** | 分享链接仅包含商家标识（企业 ID），不暴露敏感业务数据 |
| **HTTPS** | 全站强制 HTTPS，Nginx 配置 SSL 证书 |
| **CORS** | 后端配置 CORS 白名单，仅允许小程序域名访问 |

### 8.2 性能

| 需求项 | 指标 |
|--------|------|
| **首屏加载** | 冷启动 ≤ 3 秒 |
| **页面切换** | Tab 切换 ≤ 500ms |
| **列表加载** | 列表接口响应 ≤ 1 秒（100条数据以内） |
| **图片加载** | 商品图片使用懒加载 + CDN 加速 |
| **分页策略** | 列表数据分页加载，MyBatis-Plus 分页插件，每页 20 条，支持下拉刷新、上拉加载更多 |
| **蓝牙通信** | 单次打印数据传输 ≤ 5 秒（常规订单小票） |
| **Excel 处理** | 后端 EasyExcel 解析 500 行 ≤ 2 秒；前端 SheetJS 预览 ≤ 3 秒 |
| **数据库** | MySQL 连接池 HikariCP，最大连接数 20，慢查询阈值 1 秒 |

### 8.3 可用性

| 需求项 | 说明 |
|--------|------|
| **离线提示** | 网络异常时给出友好提示，缓存关键页面数据 |
| **加载状态** | 所有异步操作展示 loading 状态，避免用户重复操作 |
| **空状态** | 列表为空时展示引导性空状态页 |
| **错误处理** | 操作失败时提供明确的错误提示和重试入口 |
| **操作反馈** | 关键操作（下单、删除）需二次确认 |

### 8.4 兼容性

| 需求项 | 说明 |
|--------|------|
| **微信版本** | 支持微信 8.0 及以上版本 |
| **基础库** | 最低基础库版本 2.25.0 |
| **设备适配** | 适配 iPhone SE ~ iPhone 15 Pro Max、主流 Android 机型 |
| **屏幕适配** | 使用 rpx 单位，适配不同屏幕尺寸 |
| **蓝牙兼容** | 支持 BLE 4.0+ 热敏打印机，默认 80mm 纸宽，自适应兼容 58mm；兼容主流品牌（佳博、芯烨等） |

## 9. 数据字典（初稿）

> 以下为核心数据表的初步设计，基于 MySQL 8.x 关系型数据库。所有表包含通用字段 `id`（BIGINT 主键自增）、`created_at`、`updated_at`（时间戳）、`is_deleted`（逻辑删除）。详细 Schema 将在 Phase Two 中完善。

### 9.2 关键字段枚举值

**销售订单状态 (`oms_sales_order.status`)**：
- `PENDING` — 待确认
- `CONFIRMED` — 已确认
- `SHIPPED` — 已发货
- `COMPLETED` — 已完成
- `CANCELLED` — 已取消

**销售订单支付状态 (`oms_sales_order.payment_status`)**：
- `UNPAID` — 未支付
- `PAID` — 已支付（第一期伪支付：点击支付按钮即标记）

**采购订单状态 (`oms_purchase_order.status`)**：
- `PENDING` — 待采购
- `PURCHASING` — 采购中
- `ARRIVED` — 已到货
- `COMPLETED` — 已完成
- `CANCELLED` — 已取消

**用户角色 (`sys_user.role`)**：
- `SELLER` — 商家（企业主）
- `MEMBER` — 团队成员
- `BUYER` — 买家

### 9.3 与原 NoSQL 方案的关键差异

| 差异点 | 原方案 (NoSQL) | 新方案 (MySQL) |
|--------|---------------|---------------|
| 订单明细 | 嵌入数组 `items[]` 存在订单文档中 | 独立明细表 `_order_item`，通过 `order_id` 外键关联 |
| 商品图片 | 数组字段 `images[]` | JSON 类型字段 `images`（MySQL 8 原生支持） |
| 权限配置 | 嵌入对象 | JSON 类型字段 `permissions` |
| ID 生成 | MongoDB ObjectId | BIGINT 自增主键 |
| 订单号 | 自动生成 | 业务编码规则：`SO202603010001`（销售）/ `PO202603010001`（采购） |
